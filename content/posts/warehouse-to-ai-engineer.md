I'm not a senior engineer. I'm not a bootcamp grad. I'm a warehouse worker who decided to build his way into tech. Here's what's actually working.

## Where I Started

- Amazon fulfillment center since 2022 — pick, stow, pack
- No CS degree
- No tech job history
- Just curiosity and a willingness to break things

## What I've Done So Far

### A+ Certification (May 2026)
The foundation. Hardware, networking, troubleshooting. Not glamorous, but it proves you understand how computers actually work.

### WGU Cloud & Network Engineering
Currently enrolled. Expected completion: late 2027. The degree covers:
- Networking (CCNA track)
- Cloud infrastructure (AWS/Azure)
- Linux administration
- Automation and scripting

### The Homelab
This is where the real learning happens. I built a two-node homelab running:
- AI agents with custom MCP servers
- Container orchestration with Podman
- DNS filtering, monitoring, workflow automation
- Tailscale mesh VPN

Every post on this blog comes from real experiments in this lab.

### AI Agent Infrastructure
I run Hermes Agent — an AI that manages my homelab through custom MCP servers. 35 tools. Persistent memory. Can execute infrastructure operations through natural language.

This isn't a tutorial project. It's production infrastructure I use daily.

## The Target: AI Platform Engineer

This role didn't exist 5 years ago. Now it's one of the most in-demand positions in tech. What it requires:

| Skill | My Status |
|-------|-----------|
| Linux administration | ✅ Strong — daily driver Fedora |
| Container orchestration | ✅ Podman, learning K8s |
| Networking | ✅ Tailscale, DNS, firewall |
| Python | 🔨 Building — MCP servers, automation |
| Cloud (AWS/GCP) | 🔨 WGU coursework |
| Kubernetes | 🔨 Next major project |
| CI/CD | ✅ GitHub Actions |
| AI/ML infrastructure | ✅ Local LLMs, agent orchestration |

## What's Actually Working

1. **Build real things, not tutorials.** My homelab runs 24/7. If something breaks, I fix it. That's learning you can't get from a course.

2. **Document everything.** This blog. The wiki. GitHub repos. Every experiment becomes a portfolio piece and a learning resource.

3. **Use AI as a force multiplier.** I'm not just studying AI — I'm using AI agents to manage my infrastructure. That's the skill employers want.

4. **Get certified strategically.** A+ → Network+ → Security+ → Cloud certs. Each one opens a door.

5. **Apply before you're ready.** I applied for Amazon's OTS IT Support Engineer L4 position. Might not get it. But I learned what they're looking for.

## The Honest Part

- I still work in a warehouse
- I'm not making money from tech yet
- Some days I don't know what I'm doing
- But every week I know more than the week before

The compounding effect is real. Six months ago I'd never used Linux. Now I run a homelab with AI agents, custom MCP servers, and container orchestration.

## What's Next

- Pass the AWS Cloud Practitioner exam
- Deploy Kubernetes across both homelab nodes
- Build and deploy a production MCP server for a real business (done — KR Flooring)
- Graduate WGU (late 2027)
